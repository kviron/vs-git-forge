# Git Forge

Расширение для **VS Code** и **Cursor**, добавляющее панель с графическим интерфейсом для работы с Git: ветки, теги, история коммитов, diff и изменённые файлы.

---

## Состояние проекта

**Расширение пока очень сырое** — это ранняя версия. Реализован базовый сценарий: просмотр веток и коммитов, часть операций через контекстное меню и диалоги. Многие функции из задуманного UI (см. [docs/git-ui-blocks.md](docs/git-ui-blocks.md)) ещё не сделаны или сделаны частично. Баги и неудобства возможны; доработки планируются.

## Что уже есть

### Панель и представления

- **Git Forge** — основная webview-вкладка в нижней панели (и пункт в боковой панели Activity Bar).
- **Changed Files** — нативное дерево файлов выбранного коммита (или uncommitted changes).
- **Статус-бар** — отображает текущую ветку; клик открывает Git Forge (работает и без встроенного Git-расширения, в т.ч. в Cursor — через чтение `.git/HEAD`).

### Левая панель (webview)

- **Локальные и удалённые ветки** — дерево (Local / Remote), фильтр по поиску.
- **Теги** — список тегов репозитория.
- **Тулбар** — кнопка обновления списка веток/тегов.
- **Контекстное меню по ветке**: Checkout, New Branch from…, Checkout and Rebase onto…, Compare with…, Show Diff with Working Tree, Rebase/Merge, Push…, Rename…, Delete.
- **Контекстное меню по тегу**: Checkout, Show Diff with Working Tree, Merge into current branch, Push to origin, Delete.

### Центральная область

- **История коммитов** — список с графом веток, фильтр по ветке (в т.ч. двойной клик по ветке), по тексту/hash, по автору.
- **Строка «Uncommitted Changes»** — незакоммиченные изменения в списке.
- **Детали коммита** — сообщение, автор, дата; список изменённых файлов коммита.
- **Контекстное меню по коммиту** — создание ветки от коммита, создание тега на коммите.

### Операции и команды

- **Checkout** ветки/тега.
- **Создание ветки** — из палитры команд или из контекста (от выбранной ветки/коммита); опция «создать и переключиться».
- **Создание тега** — диалог по выбранному коммиту.
- **Pull** выбранной ветки (через кнопку Update).
- **Merge / Rebase** — из контекстного меню веток (Merge … into …, Rebase onto …).
- **Compare** — сравнение веток (терминал `git diff`).
- **Show Diff with Working Tree** — для ветки или тега.
- **Push** ветки/тега — открытие терминала с командой.
- **Rename** локальной ветки.
- **Delete** ветки (локальной/удалённой) и тега.
- **Редактирование сообщения** последнего коммита (HEAD).
- **Diff по файлу** — из дерева Changed Files открывается встроенный Diff View (два ревизии или working tree).

### Прочее

- **Локализация** — EN/RU (`l10n`, `package.nls.*.json`).
- **Lifecycle** — уведомление при установке/обновлении расширения.
- **Поддержка Cursor** — работа без встроенного Git-расширения (fallback на `.git/HEAD` и вызовы `git` по необходимости).

## Требования

- **VS Code** или **Cursor** версии `>= 1.85.0`.
- Открытая папка с Git-репозиторием (или без — тогда будет предложено инициализировать репозиторий).
- Для полной функциональности (ветки, коммиты, теги) желательно встроенное расширение **Git**; базовый показ текущей ветки в статус-баре возможен и без него.

## Установка

1. Клонируй репозиторий и открой папку в VS Code/Cursor.
2. Установи зависимости и собери расширение и webview:

   ```bash
   npm install
   npm run build:webview
   npm run compile
   ```

3. Запусти отладку: **Run and Debug** (F5) → выбери «Extension» (или конфигурацию для запуска расширения). Откроется новое окно с установленным расширением.
4. В новом окне открой папку с Git-репозиторием. В боковой панели выбери **Git Forge**, затем внизу откроется вкладка **Git Forge** с интерфейсом.

## Структура проекта

- **`src/`** — код расширения (Node.js):
  - `extension.ts` — точка входа, провайдеры webview, обработка сообщений от webview, вызовы Git API и `git`-команд.
  - `repoManager.ts` — обёртка над Git API (ветки, рефы, репозиторий).
  - `diffDocProvider.ts` — провайдер документов для встроенного Diff View.
  - `lifecycle/` — уведомления при установке/обновлении/удалении.
  - `logger.ts` — логирование.

- **`webview/`** — фронтенд панели (Solid.js + Vite):
  - `src/pages/` — страницы (главная — Git View с ветками, коммитами, тегами).
  - `src/widgets/` — виджеты (левая панель веток, история коммитов и т.д.).
  - `src/features/` — фичи (списки веток/тегов, фильтры, карточка коммита, changed files).
  - `src/entities/` — сущности (элемент ветки, тега, коммита).
  - `src/shared/` — API к расширению, типы, контекст, локализация.

- **`l10n/`** — строки расширения (EN/RU).
- **`docs/git-ui-blocks.md`** — описание задуманных блоков UI (часть ещё не реализована).

## Что сделано с помощью vibe coding (AI-ассистента)

Значимая часть кода и структуры создана и доработана в режиме **vibe coding** (разработка с помощью AI в Cursor/аналогах):

- **Контекстные меню** в webview: пункты и группы для веток и тегов (Checkout, Diff, Merge, Push, Delete и т.д.), согласование с обработчиками в `extension.ts`.
- **Локализация** — добавление и поддержка ключей в `l10n`, формулировки для диалогов и сообщений.
- **Операции с тегами** — контекстное меню тегов (checkout, diff, merge в текущую ветку, push, delete), команды `checkoutTag`, `mergeTagIntoCurrent`, `pushTag` на стороне расширения.
- **Часть UI-компонентов** и связка webview ↔ extension по сообщениям (request/response, command).
- **Документация** и заготовки под полный UI в `docs/git-ui-blocks.md`.

Ручная доработка и ревью остаются важными: не все сценарии покрыты, возможны крайние случаи и отличия в окружении.

## Известные ограничения

- Один репозиторий (первая папка workspace).
- Много операций Push/Compare и т.п. выполняются через **терминал** (команда подставляется, пользователь нажимает Enter).
- Нет stash, нет полноценного rebase UI, нет интерактивного разрешения конфликтов в интерфейсе.
- Некоторые пункты в `docs/git-ui-blocks.md` (настройки панели, группировка, избранное и т.д.) реализованы частично или не реализованы.
