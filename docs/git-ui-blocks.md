# Блоки UI интерфейса для работы с Git

Подробное описание блоков интерфейса Git-клиента (трёхпанельный макет). Все иконки левой панели работают с ветками, отображаемыми в списке веток (Local / Remote) справа от тулбара.

---

## 1. Левая боковая панель

### 1.1. Тулбар с иконками (вертикальная полоса слева)

Узкая вертикальная панель с иконками. Каждая кнопка так или иначе влияет на ветки в соседнем списке (Local / Remote).

| № | Иконка | Название | Назначение |
|---|--------|----------|------------|
| **0** | `<` (шеврон влево) | Скрыть панель | Скрывает всю левую панель вместе с панелью, на которой отображаются ветки Local и Remote. Освобождает место для центральной области. |
| **1** | `+` (плюс) | New branch | Создание новой ветки. Обычно открывает диалог или поле ввода имени новой ветки (от текущего HEAD или от выбранной ветки). |
| **2** | `↘` (стрелка обновления) | Update selected | Обновляет **выделенную** ветку в дереве веток (например, подтягивает изменения с remote для выбранной ветки или обновляет её метаданные). |
| **3** | 🗑 (корзина) | Delete branch | Удаляет **выделенную** ветку в дереве веток. Для remote-веток — удаление ссылки на удалённую ветку; для local — удаление локальной ветки. |
| **4** | `↔` (две стрелки) | Compare with current | Сравнение выбранной ветки с текущей (HEAD). Показывает разницу коммитов/файлов между текущей и выбранной веткой. |
| **5** | 🔍 (лупа) | Show my branches | Показывает только «мои» ветки — фильтр по автору/текущему пользователю, чтобы в списке оставались ветки, над которыми работаешь ты. |
| **6** | `↙` (стрелка вниз-влево) | Fetch | Выполняет **fetch** — загружает изменения с удалённого репозитория без слияния с текущей веткой. Обновляет список remote-веток. |
| **7** | ⭐ (звезда) | Mark/unmark as favorite | Отметить или снять отметку **избранного** с выделенной ветки. Избранные ветки обычно выделяются (например, звёздочкой) и могут показываться выше в списке. |
| **8** | ⏱ (часы/лог) | Navigate log to branch head | Переход в логе коммитов к **голове выбранной ветки** — центральная область (граф + список коммитов) прокручивается и выделяет последний коммит этой ветки. |
| **9** | ⚙ (шестерёнка с треугольником) | Branches pane settings | Настройки панели веток. Открывает **выпадающее меню** с опциями (см. раздел 1.2). |
| **10** | `[📁]` (папка в скобках) | Group by directory | Группировка **remote**-веток по директориям (папкам). Ветки одного remote или с общим префиксом собираются в раскрываемые папки. |
| **11** | `⮁` (два шеврона вверх-вниз) | Expand all | **Развернуть все** папки в дереве веток. Все сгруппированные remote-ветки становятся видимыми без необходимости раскрывать каждую папку. |
| **12** | `X` (скрещенные шевроны) | Collapse all | **Свернуть все** папки. Используется вместе с «Group by directory»: все группы remote-веток сворачиваются, отображаются только названия папок. |

Пункты **11** и **12** управляют раскрытием/сворачиванием папок при включённой группировке remote-веток (п. 10).

---

### 1.2. Выпадающее меню «Branches pane settings» (п. 9)

Открывается по клику на иконку шестерёнки (п. 9). Содержит настройки поведения панели веток.

- **Заголовок «On Single Click»**  
  Группа настроек: что происходит при **одиночном клике** по ветке в списке.

- **Update Branch Filter**  
  При одиночном клике по ветке — обновление фильтра веток (например, лог и граф в центре показывают только коммиты этой ветки). Может быть выбрано как действие по умолчанию (подсвечивается при наведении/выборе).

- **Navigate Log to Branch Head**  
  При одиночном клике — переход к голове выбранной ветки в логе (то же действие, что и отдельная кнопка п. 8, но вызывается кликом по ветке).

- **Show Tags** (с чекбоксом)  
  Включение/выключение отображения **тегов** в панели веток. Когда включено — в списке видны не только ветки, но и теги репозитория.

---

### 1.3. Заголовок и список веток (справа от тулбара)

- **Заголовок "HEAD (Current Branch)"** — отображает текущую активную ветку (на которой ты сейчас находишься).

- **Раздел "Local"** — дерево **локальных** веток, например:
  - `MP-8857_edit_create_project_2`
  - `master` (часто помечается звёздочкой как основная/избранная)
  - `MP-8853_add_validation` (выделенная — текущий выбор)
  - `MP-8857_edit_create_project`

- **Раздел "Remote"** — дерево **удалённых** веток (при включённой группировке — в папках), например:
  - `origin`
    - `master`
    - `DevOps`
    - `develop`
    - `stage`

Выделенная в списке ветка является объектом для действий кнопок 2–8 (обновить, удалить, сравнить, избранное, переход в лог и т.д.).

---

## 2. Верхняя панель управления и фильтров

Панель идёт слева направо: поле поиска, кнопки фильтров, выпадающие меню Branch / User / Date / Paths, иконка сортировки и опций, затем группа иконок действий справа.

---

### 2.1. Поле поиска и быстрые фильтры

- **Поле поиска «Q Text or hash»**
  - Слева — иконка лупы с маленькой стрелкой вниз (`Q`), открывающая выбор типа поиска или недавние запросы.
  - Плейсхолдер «Text or hash»: ввод строки для поиска по **тексту сообщения коммита** или по **хешу коммита** в логе.

- **Кнопка `*` (звёздочка)**  
  Включение/выключение режима (например, поиск по регулярным выражениям или фильтр по особым отметкам).

- **Кнопка «Cc»**  
  Фильтр по коммитеру (Committer) или по изменениям кода (Changes) — уточняет, по какому полю учитывать автора/изменения при фильтрации.

---

### 2.2. Выпадающее меню «Branch» (Ветка)

Открывается по клику на «Branch» (стрелка вниз). Выбор ветки, для которой показывается лог коммитов в центральной области.

- **Select...** — открывает отдельный диалог или экран для выбора ветки.
- **Favorites** — раздел избранных веток (со звёздочкой):
  - `HEAD`
  - `master`
  - `origin/master`
- **Local** — подменю с **локальными** ветками, например:
  - `master`
  - `MP-8853_add_validation`
  - `MP-8857_edit_create_project`
  - `MP-8857_edit_create_project_2`
- **origin/...** — подменю с ветками **удалённого** репозитория `origin` (список remote-веток).

---

### 2.3. Выпадающее меню «User» (Пользователь)

Фильтрация коммитов по автору.

- **Select...** — диалог выбора пользователя (по имени, email и т.п.).
- **me** — показывать только коммиты текущего пользователя.
- **Recent** — список недавно выбранных авторов для быстрого выбора.

---

### 2.4. Выпадающее меню «Date» (Дата)

Фильтрация коммитов по времени.

- **Select...** — произвольный диапазон дат (календарь или ввод дат).
- **Last 24 hours** — коммиты за последние 24 часа.
- **Last 7 days** — коммиты за последние 7 дней.

---

### 2.5. Выпадающее меню «Paths» (Пути)

Фильтрация коммитов по затронутым файлам/папкам.

- **Select...** — выбор путей (файлы или директории), по которым фильтровать лог.
- **Select in Tree...** — выбор путей в виде дерева файлов репозитория (удобно для папок и вложенных структур).

---

### 2.6. Иконка сортировки и опций (↓↑)

Кнопка с двумя вертикальными стрелками (вверх/вниз). Открывает выпадающее меню из трёх блоков:

**Sort (Сортировка):**
- **✓ By Commit Date** — сортировка по дате коммита (по умолчанию, от новых к старым).
- **Topologically** — топологическая сортировка по зависимостям в истории Git (родитель/потомок).

**Options (Опции):**
- **First Parent** — при отображении графа учитывать только первого родителя merge-коммита (упрощённая линия истории).
- **No Merges** — скрыть merge-коммиты из списка, показывать только «линейные» коммиты.

**Branch Actions (Действия с ветками в графе):**
- **Collapse Linear Branches** — свернуть линейные участки веток в графе (сделать граф компактнее).
- **Expand Linear Branches** — развернуть ранее свернутые линейные участки.

---

### 2.7. Иконки действий (правая часть панели)

- **Play** (треугольник) — воспроизведение/запуск (например, анимация истории или откат).
- **Refresh** (круговая стрелка) — обновление данных лога и веток.
- **Cherry-pick** (скрещённые веточки/ножницы) — применить выбранный коммит к текущей ветке (`git cherry-pick`).
- **Eye** (глаз) — переключение видимости элементов или режима просмотра.
- **Search** (лупа) — общий поиск или открытие панели поиска.

---

## 3. Центральная область (лог коммитов с графом Git)

- **Визуализация графа Git** — вертикальные линии и точки с ветвлениями, показывающие связи между коммитами. Зелёная точка обычно обозначает выделенный коммит или текущую ветку.

- **Список коммитов** — каждая строка:
  - сообщение коммита;
  - автор (например, "radyuk", "Радюк Роман Николаевич");
  - время/дата ("Yesterday 9:58", "05.03.2026 14:53");
  - выделенный коммит подсвечен;
  - коммиты слияния отображаются отдельно (например, "Merge branch '...' into master").

Кнопка **Navigate log to selected branch head** (тулбар, п. 8) прокручивает эту область к голове выбранной в левой панели ветки.

---

## 4. Правая боковая панель (детали коммита)

- **Верхний блок — список изменённых файлов:**
  - заголовок вида `clubm8-web 1 file` (имя проекта/репо и количество изменённых файлов в выбранном коммите);
  - дерево файлов (например, `src/components/cards/MateCard/index.tsx`).

- **Нижний блок — подробности выбранного коммита:**
  - полное сообщение коммита;
  - идентификатор, автор, email, дата (например, "b1100db3 radyuk <radyuk@...> on 06.03.2026 at 9:58");
  - иконки/метки связанных веток (`master`, `origin/master`);
  - текст вида "In 5 branches: HEAD, ... Show all" — в каких ветках присутствует этот коммит.

---

## Краткий список блоков (для навигации)

1. **Левая панель:** тулбар (0–12), заголовок HEAD, список Local, список Remote; меню Branches pane settings (On Single Click, Update Branch Filter, Navigate Log to Branch Head, Show Tags).
2. **Верхняя панель:** поиск (Q Text or hash), кнопки (*, Cc), Branch (Select, Favorites, Local, origin/...), User (Select, me, Recent), Date (Select, Last 24h, Last 7 days), Paths (Select, Select in Tree...), сортировка/опции (By Commit Date, Topologically, First Parent, No Merges, Collapse/Expand Linear Branches), иконки (Play, Refresh, Cherry-pick, Eye, Search).
3. **Центр:** граф Git + список коммитов (сообщение, автор, дата).
4. **Правая панель:** изменённые файлы + детали коммита (хеш, автор, ветки).
